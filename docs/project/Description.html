Short Description - Cat Organizer

In my application are the users able to get an overview of the information of their cat. They can add, delete and edit cats, informationlist and their informations.


Feature Description
Security

Security system using the default security plugin for Grails. The login for the admin is:

  • Username: Admin
  • Password: 1234

The login for the user is:

  • Username: User
  • Password: 1234

The admin is allowed to add new people in the household while the admin is not. Both have the same permissions for the rest of the application.
Functionality The user and the admin is able to add, edit and delete cats, lists and informations. When a cat is deleted, every list and information that belongs to the cat will be removed as well. It is not possible to delete a user, but it is possible edit it and the admin also can add new users.
User Interface

I made a color-concept and I designed every icon and button by myself. The website is responsive.
Dependencies

I only added the security plugin we also used in lecture.
Code

I only used the documentation (https://grails.org/) and the material we discussed in lecture. I also discussed a few things with other webec students. The security is generated by Grails.
Tests

I tested my validations and two methods. Because I used hasMany and belongsTo, it was really difficult to write test for the relations
